The MSDK contains a large number of libraries, both in-house and third-party. The primary library is the [Peripheral Driver API](/libraries-peripheral-drivers), which provides a higher-level C API over hardware registers. The MSDK also ships dedicated libraries for more complex internal hardware peripherals such as USB controllers, the SDHC interface, and the Cordio BLE stack. Additional drivers for external components — TFT displays, cameras, accelerometers, and audio codecs — are available in the Miscellaneous Drivers collection.

## Enabling libraries

Libraries are enabled per-project using `LIB_*` build variables in your `project.mk` file. Each variable accepts `0` (disabled) or `1` (enabled).

```makefile
# project.mk

LIB_FREERTOS = 1
LIB_CORDIO   = 1
LIB_MAXUSB   = 1
```

The full list of toggle variables is shown below. Some libraries expose additional configuration variables once enabled; see each library's page for details.

| Variable | Library |
|---|---|
| `LIB_PERIPHDRIVERS` | Peripheral Driver API (enabled by default) |
| `LIB_BOARD` | Board Support Package (enabled by default) |
| `LIB_CMSIS_DSP` | CMSIS-DSP |
| `LIB_CORDIO` | Cordio Bluetooth Low Energy |
| `LIB_FCL` | Free Cryptographic Library |
| `LIB_FREERTOS` | FreeRTOS |
| `LIB_LC3` | LC3 audio codec |
| `LIB_LITTLEFS` | littleFS filesystem |
| `LIB_MAXUSB` | MAXUSB |
| `LIB_TINY_USB` | TinyUSB |
| `LIB_SDHC` | SDHC / FatFS |
| `LIB_CLI` | CLI library |
| `LIB_UNITY` | Unity test framework |
